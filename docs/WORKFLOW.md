# Development Workflow Guide

## Startup Checklist
1. Lire `.github/copilot-instructions.md`
2. Lire `docs/PROJECT_CONTEXT.md` et `docs/ARCHITECTURE.md`
3. Vérifier la branche active (`main` suivi) et la tâche courante

## Task Execution
- Plan: résumer la demande, lister les fichiers, vérifier impacts.
- Implémentation: suivre le style (2 espaces, single quotes), garder CTA Next sticky et thème clair.
- Test: au minimum lancer l’app (`npx expo start`) ou vérifier la logique modifiée.

## After Work
- Résumer les modifications et tests effectués.
- Mettre à jour les logs pertinents (Recent Changes dans copilot-instructions si gros changement).

## Troubleshooting
- Conflits: décrire et demander arbitrage.
- Contexte flou: poser une question ciblée.
- Régression UI/UX: vérifier accessibilité (CTA atteignable, contrastes suffisants).
